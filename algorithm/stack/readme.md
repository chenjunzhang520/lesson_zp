## 数组和链表实现栈的优缺点
- 时间效率
  入栈和出栈操作都在预先分配好的连续内存中进行， 效率高
  如果入栈时超出数组容量， 会触发扩容机制，导致入栈操作时间复杂度为O(n)
  复制所有元素到新的数组中， 时间复杂度为O(n)
  链表虽然在入栈的时候开销大一些 (ListNode实例化)， 但扩容非常灵活，O(1)
  - 基于数组实现的栈在触发扩容时效率会降低到O(n)，但由于扩容是低频操作，因此平均效率高。
  - 基于链表的栈可以实现更加稳定的效率表现。
- 空间效率
  基于数组的栈可能造成一定的空间浪费。由于链表节点需要额外的空间来存储指针，因此链表节点占用的空间相对较大。
