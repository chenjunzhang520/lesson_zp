# Agent 智能体 ？
字面意义上，AI会自己干活。
这么理解没问题，但遮住了一个更重要的问题。

Agent和普通的AI对话 到底它的差别在哪里呢？
结构导致的 Agent 工作方式
- 对话结构
普通对话，你问它答，一问一答就结束。
案例 帮我写一封邮件
llm 写完，任务就终止了。
问答机器，输出一次，没有后续。
Agent 不一样，他又一个持续运转的结构。
你给他一个任务 他会去拆任务，决定下一步，去调工具，看结果。这过程是一个循环，直到任务完成，或者它自己判断(超出循环次数、token上限、相同结果次数，失败了)

## Agent工作方式， 核心三个动作
- 第一个动作 思考  Reason 
- 第二个动作 行动  Act
- 第三个动作 观察  Observe

一轮的，观察后再回到思考，再行动，再观察，循环往复，这个循环，就叫做ReAct。
ReAct (Reasoning + Act + Observe) 是一套标准的Agent 执行流程/Agent工作框架，不是langChain那种大型开发库，是Agent 通用的循环工作标准。

## demo
帮我写分析的竞品，然后去写一份报告。Agent会怎么做。

- 第一轮它先去思考 (reason)
    需要去搜索竞品的信息 
    调用搜索工具(act)，去咨询三家竞品的最新动态 
    查完以后，把结果拿回来。
    然后观察 (observe) 信息量挺大。
    到这里第一轮结束了，接着？

    再回到思考，发现缺少财务数据(reason)
    可以去官网或查股市API(Act)
    去抓取相应数据
    观察
    第三轮、第四轮....
    最后一轮，把报告写完，交给我们。

Agent 最核心的动作，就是Tool Use
工具是Agent的手和脚，没有工具，他只能在脑子里转。转完之后，还是只有文字。
常见的工具：
- 搜索工具，能上网查实时信息
- 代码执行器，能运转代码，能看结果。
    Anthopic 最牛逼的Agent 企业，代码是最标准，又完善工程化验收机制的，ai 测试agent 
    文无第一，武无第二 (claude)
- 文件读写I/O 
- 浏览器的操控browser 打开网页，点击提交，
    manus 
- API 调用 
工具越多， Agent能干的事情越多。
很多Agent产品，调用了很多工具，工具的覆盖范围。直接决定Agent的能力边界，选择Agent的核心。
