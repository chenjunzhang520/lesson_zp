WorkFlow 本质上是一套预定义的流程，可以把它想象成一套生产的流水线。每一步都有明确的输入和输出。每个节点的逻辑是固定的，当条件满足，就会进入下一环节。
在技术上，它通常变成一个编排好的工作流引擎，
一个由AI调用，条件判断，逻辑循环组成的一个图谱。

langchain 适用于快速搭建工作流的开发框架，灵活构建各类智能任务链路

## 思考
AI 工作流也能干活， 他和Agent（reAct）有何区别？
很多人会混淆，反正都是AI 在执行任务

## Agent 智能体
和workflow的逻辑完全不一样，他不走在预设的轨道上，而是站在开放的空间里，自己去探索怎么去做（智能）
从技术角度，一个Agent 往往具备三个大的核心能力

第一个能力：感知环境
通过输入文本、图像、数据来理解任务。由哪些工具可以调用，当前情况如何。如何分步骤，如何规划
第二个：规划路线  
！！！不是预先定义好的，而是动态去生成任务链。
第三个：执行任务
去调用工具，完成子任务，并不断去调整。

## 举例
你让Agent 去规划旅游、他可能会先去搜一下酒店和机票价格，再去查天气和签证需求，然后结合你的偏好和预算，动态去权衡方案。整个过程，他可能走的路径和你预期的完全不同，这就是Agent的关键区别，自主性和适应性
它不会被一个固定的流程限制，而是像一个思考中的个体。所以，本质的区别在于：workflow就好像一条高速公路，路线很清晰很固定，效率很高，但能到达既定的重点。
Agent 更像是一个司机，他知道要去哪 但可以随时去改道(修路，感知环境)，尝试新的路径，甚至在目的地发生变化时候，自行调整。

所以，从产业角度，workflow 更适合流程化，重复性的任务。
比如审批流程、数据清洗、客服AI
Agent 更适合动态的一个角色场景。比如：复杂程度、策略规划。

从技术栈的领域，workflow 更依赖于，图形化编排（coze），流程引擎。Agent更易俩与大模型的推理能力和记忆能力，包括他的工具调用等。

未来我们的AI 工程中将会集合Agent 和 workflow 的双重能力。
底层是workflow 去提供稳定性和可控性。上层是Agent 提供灵活性和智能性

workflow是骨架，Agent 是Workflow的大脑
workflow 没什么创造力，Agent 有，但是不可控，不可预测，一起结合，既高效又有智慧。
