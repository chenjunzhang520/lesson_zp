# 轻松上手 Vibe Coding

想去用AI生成写代码， 但AI 生成的要不是"幻觉代码"，看起来像那么回事，一跑就出错。要么就是屎山代码， 能跑， 但乱的你想改都不知道从哪下手。

这个指南，不是怎么写代码，而叫我们去与AI 协作写出靠谱的代码

- 新手 自然语言编程
- 不要急于写代码，先思考，规划
  每次prompt 都会自动带上规划，保持一致性，有整体的约束

## 操作核心方法
- 第一步： 规划就是一切。
不要让AI去写代码，先强制它去读懂你的技术栈和项目规划。(/init)
就好像你不会让新来的员工直接干活，去阿里入职 阿里Java 员工手册，等，
得让它熟悉公司业务流程一样，AI 生成代码逻辑清晰多了

 帮我写一个React 代办清单页面， 支持新增、删除任务。

 - 先规划，再编码
 ```
 遵守胶水编程思维: 优先使用成熟方案，避免凭空造逻辑
 第一个阶段: 只做规划，禁止输出任何代码。
 1. 确认技术栈:
 React 19 + TailWindcss + useState
 2. 梳理功能边界
 - 新增待办、删除代办、切换完成状态
 - 不做本地持久化、筛选、拖拽等功能
 3. 拆分模块 (乐高组件)
 输入框组件、待办条目组件、列表容器组件
 4. 定义数据流
 useState 存储task 数组 数据结构:
 {id, text, completed}
 5. 输出这份完整规划，等待我确认无误后，再分段实现代码
 ```

背后逻辑
先划定边界; 防止AI 擅自多加功能，无限膨胀成屎山。
强制模块拆分 (好读好维护，AI生成代码我们都要审核)，
引导AI复用独立组件，预定规定数据结构，从根源减少AI 参数、字段幻觉 (title，不能是text，字段不能让AI 做主)。
先生成规划、审核规划、更新规划。
这份规划将伴随我们的项目开发完整周期。

第二步: "胶水"编程思维
能抄不写（github 经典，优质，经过验证的代码），能连不造（A、B、C 组件 胶水）。指挥AI 去找那些已经被验证过千万次的成熟组件，拼乐高一样，把它组装起来。
胶水 比喻， 可以粘和， 少量 （AI生成的）， 产生幻觉和屎山的可能性大大降低。

胶水本身不创造零件，只负责把现成零件粘在一起，你只负责写衔接、调用、流转的粘合代码，把各个模块联通。

轮子别人造好，你只做胶水; 胶水不生成零件，只联通零件。

## 举例
React 待办 连贯演示。
我们已经完成项目的规划、现在正式写代码，对比两种方式:

- 错误示范 (指令)
  从零造零件，高幻觉风险
  帮我写React待办清单的拖拽排序功能 (秀)
  AI很可能凭空手写一套拖拽逻辑，自行实现坐标监听，排序算法。
  问题: 手写拖拽逻辑边界case 很多(规划写的复杂)，容易出现幻觉bug，代码难以维护。

  遵守胶水编程原则: 绝不从零自研底层逻辑，优先使用长期验证成熟开源组。
  当前需求: 给待办列表增加拖拽排序。
  1. 先调研: React生态成熟的拖拽库，优先选用react-beautiful-dnd(业内广泛使用)
  npm install react-beautiful-dnd
  <react-beautiful-dnd>
  ... list 
  </react-beautiful-dnd>
  2. 不要自己手写拖拽底层代码，只做粘合工作。
  3. 输出内容顺序:
  安装依赖命令、把现有TodoList组件和react-beautiful-dnd 进行衔接
  只写模块之间适配、数据流转的粘合代码

cc/codex 记忆模块、harness 架构
可以不断的帮我们优化提示词
